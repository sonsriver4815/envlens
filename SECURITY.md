# Security Policy

configenvy checks environment variable usage, examples, documentation, and CI configuration. Please do not include real secrets, tokens, private keys, production URLs, or other sensitive values in public issues or pull requests.

## Reporting a vulnerability

If you find a vulnerability in configenvy, please report it through GitHub's private vulnerability reporting or contact the repository owner directly through GitHub.

Include:

- A short summary of the issue.
- Steps to reproduce the behavior.
- The affected configenvy version.
- Any safe sample input needed to understand the issue.

Please use redacted examples such as `sk_test_***` or `example-token` instead of real credentials.

## Supported versions

Security fixes target the latest published version of configenvy.
